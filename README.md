# Table CRUD Test

Тестовое приложение для управления данными в таблице с возможностью создания, чтения, обновления и удаления записей (CRUD).

## 🛠 Технологический стек

- **Frontend:** React (Vite)
- **Язык:** TypeScript
- **Стилизация:** SCSS Modules
- **Состояние:** React Hooks (useState)
- **Линтинг и форматирование:**
  - **ESLint** — для поиска ошибок и поддержания чистоты кода.
  - **Simple Import Sort (плагин ESLint)** — для автоматической группировки и логической сортировки импортов (сначала библиотеки, затем внутренние модули и стили).
  - **Prettier** — для единообразного форматирования кода.

## 🚀 Как запустить проект

1.  **Клонируйте репозиторий:**

    ```bash
    git clone https://github.com
    cd table-crud-test
    ```

2.  **Установите зависимости:**

    ```bash
    npm install
    ```

3.  **Запустите проект в режиме разработки:**
    ```bash
    npm run dev
    ```

## 🧹 Качество кода (ESLint & Prettier)

В проекте настроены инструменты для автоматической проверки и форматирования кода.

- **Проверка линтером:**
  ```bash
  npm run lint
  ```
- **Автоматическое исправление ошибок форматирования:**
  ```bash
  npm run format
  ```

## 📂 Структура проекта (Архитектура FSD)

- `src/app` — инициализация приложения: провайдеры, глобальные стили.
- `src/features` — функциональные части: логика (в т.ч. CRUD) и действия пользователя.
- `src/entities` — бизнес-сущности: UI, константы и модели данных.
- `src/shared` — базовый UI, утилиты и конфиги (без привязки к бизнесу).
- `.eslintrc.js` — конфигурация правил ESLint.
- `.prettierrc` — настройки форматирования Prettier.
